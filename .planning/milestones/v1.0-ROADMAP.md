## Overview

GSD Monitor already has a working Python/FastAPI/React skeleton — these phases fix what is broken and add what is missing. Phase 1 eliminates the duplicate-project problem (the #1 user complaint), Phase 2 replaces the visual structure with one that makes status legible at a glance, Phase 3 adds the doc browser so every planning file is readable from within the app, and Phase 4 closes the backend correctness and performance gaps that remain.

## Key Accomplishments

1. **Worktree Deduplication** — Each canonical git repo appears exactly once in the project dropdown, with a badge showing worktree count and a CSS-only hover tooltip listing branch/directory names
2. **VS Code Dark Dashboard** — Stats bar (% complete, phases done/total, active phase), breadcrumb, and color-coded phase list all visible above the fold without any click
3. **Full Doc Browser** — All `.planning/` files navigable in a two-column layout; ROADMAP.md, STATE.md, active PLAN.md, and REQUIREMENTS.md surfaced as quick-access shortcuts; any file renders as inline markdown
4. **Performance & Correctness** — Non-blocking trylock coalesces FS events, `_EXCLUDED_DIRS` skips `node_modules`/`.venv`/`build`/`dist`/`.git`, `StateParser` wired for both GSD-1 and GSD-2 projects, settings save race condition eliminated
5. **FastAPI & Python Modernization** — `@on_event` replaced with lifespan context manager; all 9 `datetime.utcfromtimestamp()` calls replaced with timezone-aware `datetime.fromtimestamp(ts, tz=timezone.utc)`
6. **Frontend Source & Bundle Rebuilt** — `context.tsx`, `SettingsPage.tsx`, stub pages for v2 features (`DriftPage`, `QuickTasksPage`, `VerificationPage`) committed to master; fresh `frontend/dist` bundle built with zero TypeScript errors
