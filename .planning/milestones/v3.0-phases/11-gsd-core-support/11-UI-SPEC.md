## Component Inventory

New UI elements required by Phase 11 requirements:

### 1. Progress Bar (PROG-02)
- **Dashboard card compact variant:** Full-width bar inside existing stats card. Height 4px. Background #474747, fill #4ec994 (green) scaled to completion %. No label inside bar.
- **Project detail header fuller variant:** Stacked layout — "X of Y phases complete" label (text-xs text-[#858585]) above a full-width bar at 8px height. Percentage displayed inline: "42%".
- Both variants appear only when `progress_percent`, `completed_phases`, and `total_phases` are available from STATE.md (PROG-01, PROG-03).

### 2. Pause Status Banner (DOCS-06, DOCS-07)
- Location: Below breadcrumb in DashboardPage, above the stats grid, only visible when `paused: true` in HANDOFF.json.
- Appearance: `rounded-md border border-amber-800/40 bg-amber-900/20 px-4 py-2` containing:
  - Left: amber-400 dot (4px circle) + "Paused" label (text-xs font-semibold text-amber-400)
  - Right: "Phase {N}, Plan {N} · paused {date}" (text-xs text-[#858585])
- No CTA — app is read-only. No dismiss button.
- Resume context (.continue-here.md) accessible via Docs page; link not needed in banner.

### 3. Config Summary Badge Row (DOCS-08)
- Location: Below GSD version badge in DashboardPage, one row of small badges.
- Shows three properties from config.json: workflow_mode, model_profile, branching_strategy.
- Each badge: `rounded bg-[#2a2d2e] px-1.5 py-0.5 text-xs text-[#858585]`
- Only rendered when config.json is present (gsd-core projects only). Hidden for GSD-1 legacy projects.

### 4. Quick-Access Shortcuts for New Doc Types (DOCS-01 through DOCS-05)
- Location: DocsPage quick-access list (currently shows ROADMAP.md, STATE.md, active PLAN, REQUIREMENTS.md).
- Add to the list when present: VERIFICATION.md (phase-level), UI-SPEC.md (phase-level), UI-REVIEW.md (phase-level), SUMMARY.md (plan-level).
- Phase-level shortcuts label format: `{DOCTYPE} ({phase-number-padded})` — matches existing `PLAN (05)` pattern.
- Only append shortcuts for documents that exist on disk (use has_* flags on PhasePayload).
- No new icons; use existing inline-block spacer pattern.

### 5. Phase ID Display (DETECT-04)
- Milestone-prefixed phase IDs (e.g. "Phase 1-01") displayed as-is in phase number column.
- Use string rendering instead of `String(p.number).padStart(2, "0")` when `p.code` is present.
- Display: if `p.code` is present → use `p.code`, else → `String(p.number).padStart(2, "0")`.
- Existing font-mono pattern maintained.

### 6. gsd-core Version Badge (DETECT-01)
- Replace the existing `stats?.gsdVersion === "gsd2" ? "GSD-2" : "GSD-1"` ternary.
- New logic: `version === "gsd-core"` → "gsd-core" badge with bg-[#007acc]/30. `version === "gsd1"` → "GSD-1" badge (neutral bg-[#2a2d2e]). No GSD-2 badge (D-10, D-11).

## Interaction Contracts

### Progress Bar
- Not interactive (display only). No click, no tooltip in compact variant.
- Detail variant: no interaction. Static render.

### Pause Banner
- Not interactive. No dismiss. No navigation link.
- Visible only when backend provides `paused: true` from HANDOFF.json.
- Hidden when project is not paused.

### Config Badge Row
- Not interactive. Display only. No expand/collapse.

### Quick-Access Shortcuts (new doc types)
- Same click behavior as existing quick-access: calls `handleFileClick(path)` → loads content into main panel.
- Files absent from disk: item is omitted from quick-access list entirely (no disabled state).

### Phase Cards (milestone-prefixed IDs)
- Same click behavior as existing phase cards → opens drawer.
- Phase ID display change is visual only; interaction contract unchanged.

### GSD-2 Removal (D-10, D-11)
- No frontend interaction change needed — GSD-2 version checks and V2 segment conditionals are deleted.
- Any segment with `gsdVersion === "gsd2"` will no longer appear in groups payload from backend.
- Frontend cleanup: remove all `gsdVersion === "gsd2"` conditionals from ShellLayout.tsx segment selector.
