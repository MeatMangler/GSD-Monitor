# Quick Task 260405-759: Fix stale data in the Projects/Docs UI — Summary

**One-liner:** DocsPage now auto-refetches tree and content on WebSocket push via refreshKey, and refresh() re-scans when _dirty was set during a concurrent scan.

## Completed Tasks

### Task 1: DocsPage — refetch tree and content on project data refresh

**File:** `frontend/src/pages/DocsPage.tsx`

- Added `useRef` to the React import (alongside existing hooks)
- Added `refreshKey` state (integer counter) initialized to 0
- Extracted `groups` from `useApp()` alongside existing destructured values
- Added `isInitialMount` ref to skip the first groups change (avoids spurious refetch on mount)
- Added `useEffect` that increments `refreshKey` on every new `groups` reference (which changes on every `reload()` call in context.tsx)
- Added `refreshKey` to the dependency array of the tree-fetch effect (line ~80) and the file-content-fetch effect (line ~95)
- Did NOT touch the reset effect keyed on `planningPath` — that correctly resets selection state only on project switch

**Effect:** When WebSocket fires `projects_updated` -> context.tsx calls `reload()` -> `groups` reference changes -> `refreshKey` increments -> both fetch effects re-run with fresh data from API.

### Task 2: Backend — re-scan on _dirty after refresh()

**File:** `gsd_monitor/api/app.py`

- Added `self._dirty: bool = False` field to `RuntimeState.__init__`
- Updated `_on_fs_change`: when lock acquire fails (scan in progress), sets `self._dirty = True` instead of silently dropping the event
- Replaced `refresh()` single-scan body with a `while True` dirty-loop pattern:
  - Clears `_dirty = False` at loop start
  - Runs `discover_groups`
  - If `_dirty` was set during the scan (by a concurrent watcher callback), loops again
  - Breaks when scan completes without a new dirty signal
- Moved `watcher.set_roots(...)` outside the loop — only needs to run once per refresh call

**Effect:** Filesystem events that arrive while `refresh()` holds the lock are no longer silently dropped. The scan reruns until it captures a clean state.
