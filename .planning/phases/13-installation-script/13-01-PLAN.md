  <action>
Create `install.ps1` at the repo root. The script is a single-file, linear PowerShell installer with the following sections in order:

**Script header:**
- `#Requires -Version 5.1` directive
- `$ErrorActionPreference = 'Stop'`
- Accept optional parameters: `[CmdletBinding()] param([string]$InstallPath, [switch]$NoWait)` — `$InstallPath` overrides the default install directory; `$NoWait` skips the final "Press any key" pause
- Define helper functions: `Write-Step` (cyan step announcements), `Write-OK` (green confirmations), `Fail` (red error + exit 1)
- Print a banner: "GSD Monitor Installer" with a separator line

**Section 1 — Prerequisite checks (per D-07, D-08):**
All checks run upfront before any filesystem mutation.

- Git: `git --version 2>&1`. If `$LASTEXITCODE -ne 0` or null output, call `Fail` with message naming Git and URL `https://git-scm.com/download/win`
- Python 3.11+: `python --version 2>&1`. Parse output with regex `Python (\d+)\.(\d+)`. Compare major/minor. If missing or below 3.11, call `Fail` with message naming Python and URL `https://www.python.org/downloads/`
- Node.js: `node --version 2>&1`. If `$LASTEXITCODE -ne 0` or null, call `Fail` with message naming Node.js and URL `https://nodejs.org/`
- WebView2: Check BOTH registry paths (per D-08 and RESEARCH Pattern 3):
  `HKLM:\SOFTWARE\WOW6432Node\Microsoft\EdgeUpdate\Clients\{F3017226-FE2A-4295-8BDF-00C3A9A7E4C5}`
  `HKCU:\Software\Microsoft\EdgeUpdate\Clients\{F3017226-FE2A-4295-8BDF-00C3A9A7E4C5}`
  Read the `pv` property from each. WebView2 is present if any `pv` value is non-null, non-empty, and not `0.0.0.0`. If absent, call `Fail` with message and URL `https://developer.microsoft.com/microsoft-edge/webview2/consumer/`

Each successful check prints via `Write-OK` showing the detected version string.

**Section 2 — Install path resolution (per D-02):**
- Default: `Join-Path $env:LOCALAPPDATA 'GSDMonitor'` (user-writable, no UAC — per RESEARCH anti-pattern advice)
- If `$InstallPath` parameter was provided, use it directly; otherwise prompt with `Read-Host "Install directory [$defaultPath]"` and fall back to default if input is empty
- Print the resolved path

**Section 3 — Upgrade detection (per D-04):**
- Check `Test-Path (Join-Path $installDir '.git')`
- If existing install found:
  - Run `git -C $installDir log --oneline -1` to get current commit
  - Run `git -C $installDir fetch origin 2>&1 | Out-Null` then check `$LASTEXITCODE`
  - Run `git -C $installDir log --oneline -1 origin/HEAD` to get remote commit
  - Display both with labels "Current:" and "Latest:"
  - Prompt `Read-Host "Upgrade GSD Monitor? [Y/n]"`
  - If answer matches `^[Nn]`, print "No changes made." and `exit 0`
  - On yes: `git -C $installDir pull` with `$LASTEXITCODE` check
  - Rebuild frontend: `Push-Location` to `$installDir\frontend`, run `npm install` then `npm run build`, each with `$LASTEXITCODE` check, `Pop-Location`
  - Also re-install pip dependencies (in case new ones were added): `& "$installDir\.venv\Scripts\pip.exe" install fastapi "uvicorn[standard]" pydantic pydantic-settings pywebview pygit2 watchdog` with `$LASTEXITCODE` check
  - Print "Upgrade complete." in green and exit 0

**Section 4 — Fresh install (per D-01):**
Only reached if no existing `.git` found at install path.

- Git clone: `git clone https://github.com/MeatMangler/GSD-Monitor.git $installDir` with `$LASTEXITCODE` check (repo URL confirmed from `git remote -v` in RESEARCH.md)
- Python venv: `python -m venv "$installDir\.venv"` with `$LASTEXITCODE` check. Do NOT run Activate.ps1 (per RESEARCH anti-pattern).
- pip install via absolute path: `& "$installDir\.venv\Scripts\pip.exe" install fastapi "uvicorn[standard]" pydantic pydantic-settings pywebview pygit2 watchdog` with `$LASTEXITCODE` check. This is the exact package list from README.md line 55.
- Frontend build: `Push-Location (Join-Path $installDir 'frontend')`, run `npm install` then `npm run build`, each with `$LASTEXITCODE` check (on failure: `Pop-Location` before `Fail`), then `Pop-Location`

**Section 5 — Launcher creation (per D-05):**
- Write `start.bat` into `$installDir` using `Set-Content` with `-Encoding ASCII`. The exact content (verified from the repo `start.bat`):
  Line 1: `@echo off`
  Line 2: `cd /d "%~dp0"`
  Line 3: `start "" /b .venv\Scripts\pythonw.exe -m gsd_monitor`

**Section 6 — Desktop shortcut (per D-06):**
- Get desktop path: `[Environment]::GetFolderPath('Desktop')`
- Shortcut file: `Start GSD Monitor.lnk` on the desktop
- Create via `New-Object -ComObject 'WScript.Shell'` then `CreateShortcut`
- Set `TargetPath` to `Join-Path $installDir 'start.bat'`
- Set `WorkingDirectory` to `$installDir`
- Set `Description` to `GSD Monitor — project status dashboard`
- Set icon: if `Test-Path (Join-Path $installDir 'assets\gsd-icon.ico')`, set `IconLocation` to that path with `, 0` suffix
- Set `WindowStyle = 7` (minimized — the bat window should not stay visible; the pywebview window is the real UI)
- Call `.Save()`

**Section 7 — Success message and optional pause:**
- Print success banner in green: "GSD Monitor installed successfully!"
- Print the install location
- Print: "Launch via the desktop shortcut 'Start GSD Monitor' or run: {installDir}\start.bat"
- If NOT `$NoWait`: print blank line then `Read-Host "Press Enter to exit"` (more reliable than `pause` in PowerShell)

**Error handling throughout:**
- Every native command (git, python, node, npm, pip) MUST be followed by `if ($LASTEXITCODE -ne 0) { Fail "..." }` — $ErrorActionPreference does NOT catch native command failures (per RESEARCH Pitfall 1)
- Wrap the try blocks around each tool check with `catch { $var = $null }` to handle commands not on PATH
- Do NOT call `Set-ExecutionPolicy` anywhere in the script (per RESEARCH anti-pattern)

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub raw -> user PowerShell | User executes remotely-fetched script via `iex (irm URL)` — trusts GitHub HTTPS |
| User input -> filesystem | Install path from `Read-Host` used in `git clone` and file writes |
| npm/pip registries -> install dir | Package managers fetch from public registries during install |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Spoofing | iex (irm URL) delivery | accept | Script hosted on GitHub over HTTPS; user explicitly chose this pattern (D-03); script is public and inspectable |
| T-13-02 | Tampering | Install path input | mitigate | Resolve path to absolute; reject paths containing `..` segments; use only within `$installDir` variable |
| T-13-03 | Tampering | npm/pip package installs | accept | No new packages introduced — same packages as existing README instructions; supply chain risk is pre-existing |
| T-13-04 | Elevation of Privilege | Script execution context | mitigate | Script does NOT request UAC elevation; default path is user-writable `$env:LOCALAPPDATA`; no `Set-ExecutionPolicy` calls |
| T-13-SC | Tampering | npm/pip installs | accept | No new packages introduced in this phase; existing project packages only; no package legitimacy audit needed |
</threat_model>
