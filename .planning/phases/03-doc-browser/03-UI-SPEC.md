# Phase 3 — UI Design Contract

> Visual and interaction contract for Phase 3: Doc Browser.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — Tailwind CSS v4 via `@tailwindcss/vite` plugin |
| Preset | not applicable |
| Component library | none — inline Tailwind utility classes only |
| Icon library | none — text labels and Unicode characters only |
| Font | Segoe UI, system-ui, sans-serif (declared in `index.css`) |

No shadcn. No `tailwind.config.js`. All design tokens are VS Code hex values applied as Tailwind arbitrary values (e.g., `bg-[#1e1e1e]`). This is the established project pattern from Phase 2.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tree node indent per depth level (`pl-4` = 16px per level is the standard; `gap-1` = 4px for icon-to-label gaps) |
| sm | 8px | Tree node vertical padding (`py-1` = 4px top+bottom, `px-2` = 8px sides) |
| md | 16px | Default element spacing; content pane padding sides |
| lg | 24px | Content pane padding (`p-6` = 24px, matching DashboardPage) |
| xl | 32px | Section breaks in content pane |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Tree panel width: 260px fixed (`w-[260px]`) — not a spacing token, a layout dimension. Source: CONTEXT.md D-02 ("~240–280px fixed"); 260px is the midpoint.
- Tree item touch target minimum: 28px tall (`min-h-7`) — ensures clickable rows meet minimum usability threshold.
- Quick-access section label: `px-2 py-1` = 8px horizontal, 4px vertical — muted section header, not a clickable item.

## Typography

Declared weights: **2** — 400 (regular) and 500 (medium). This satisfies the 2-weight maximum.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 regular | 1.5 |
| Label / tree node | 12px (`text-xs`) | 500 medium | 1.4 |
| Section header (Quick access, Files) | 11px (`text-xs` + `uppercase tracking-wider`) | 500 medium | 1.4 |

Notes:
- Section headers (11px, uppercase, `tracking-wider`) are visually distinguished from tree nodes (12px, no transform) by size, case, and letter-spacing — not by weight. Both use 500.
- Prose heading weights (600/700) are `prose-sm` plugin defaults, not tracked as design tokens. The `prose prose-invert prose-sm max-w-none` class manages its own internal heading hierarchy automatically.
- Tailwind v4 `text-xs` renders at **12px** (0.75rem). Use `text-xs` in code; do not use `text-[13px]`.
- Monospace font applied to file/folder names via `font-mono` — inherits browser monospace stack. Source: established in DashboardPage.tsx phase number rendering (`font-mono font-medium`).
- `prose prose-invert prose-sm max-w-none` for rendered markdown — identical to existing Drawer.tsx and DashboardPage.tsx patterns. This is non-negotiable; `prose-invert` is required for dark-background legibility.
- Raw `<pre>` blocks for non-markdown files: `text-sm text-[#cccccc] whitespace-pre-wrap` — inherits body font.

| Component | Location | Reuse Pattern |

| `DocsPage` | `frontend/src/pages/DocsPage.tsx` | New page component, named export |

### Tree node visual states

| State | Background | Text color |
|-------|------------|------------|
| Default (file) | transparent | `text-[#858585]` |
| Hover (file) | `hover:bg-[#2a2d2e]` | `hover:text-[#cccccc]` |
| Selected (file) | `bg-[#2a2d2e]` | `text-[#007acc]` |
| Directory (collapsed) | transparent | `text-[#858585]` |
| Directory (expanded) | transparent | `text-[#cccccc]` |
| Directory hover | `hover:bg-[#2a2d2e]` | `hover:text-[#cccccc]` |

### Directory expand/collapse indicator

Use a plain Unicode chevron: `›` (collapsed) / `⌄` (expanded), rendered as `text-xs text-[#858585] mr-1 select-none`. No SVG icon library. Width is fixed at one character to prevent layout shift on toggle.

### Filename truncation

Long filenames: `truncate max-w-full` on the filename `<span>`. No tooltip — truncation is acceptable for v1. Applies to both tree nodes and quick-access items.

## Interaction Contract

### Clicking a file node

1. Set `selectedPath` to the file's relative path (forward-slash normalized).
2. If the file is already selected: no-op (do not re-fetch).
3. Trigger content fetch (`fetchDocFile`).
4. During fetch: show "Loading…" in content pane.
5. On success: render content per file type (markdown or `<pre>`).
6. On error: render error copy in content pane.

### Clicking a directory node

1. Toggle directory in `expandedDirs` set.
2. No content fetch — directories are not selectable.
3. No visual selection indicator on directories.

### Clicking a quick-access item

- Identical behavior to clicking the same file in the full tree.
- Both the quick-access item and the corresponding full-tree node must reflect the selected state simultaneously (both keyed by the same `selectedPath` string).

### Project change (`selectedGroupId` changes)

1. Reset `selectedPath` to `"ROADMAP.md"`.
2. Reset `expandedDirs` to empty set.
3. Re-fetch tree for the new `planningPath`.
4. Auto-fetch ROADMAP.md content.
5. Source: CONTEXT.md D-19.

### On page mount (initial load)

1. Auto-select `"ROADMAP.md"` as `selectedPath`.
2. Fetch tree.
3. Fetch ROADMAP.md content.
4. Source: CONTEXT.md D-10.

### Non-.md file rendering

All files not ending in `.md` render as `<pre>` with raw content. Source: CONTEXT.md D-11, D-13.

## Accessibility Notes

- All clickable tree nodes must be `<button type="button">` elements, not `<div onClick>`. This ensures keyboard focus and `Enter`/`Space` activation work without additional event handling.
- Directory toggle buttons must have an `aria-expanded` attribute reflecting the expanded state.
- Selected file must have `aria-current="true"` on its button element.
- Content pane must have `role="region"` and `aria-label="Document content"`.
- Tree panel must have `role="navigation"` and `aria-label="Planning files"`.
- Minimum click target: 28px height (`min-h-7`) on all tree nodes.
